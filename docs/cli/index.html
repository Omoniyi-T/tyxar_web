Command Line Interface (CLI) Documentation

Overview

Tyxar represents a paradigm shift in programming language design—an English-first programming language that fundamentally reimagines how developers interact with code. By leveraging natural language constructs and eliminating cryptic symbolic syntax, Tyxar enables you to write code that reads almost exactly like you speak.

The Tyxar Command Line Interface (CLI) provides a comprehensive suite of tools for compiling, executing, transpiling, and debugging your applications. Each command has been meticulously designed for discoverability and intuitive usage, whether you're a first-time programmer or an experienced software engineer.

File Extension and Basic Syntax

All Tyxar source files utilize the distinctive .txr file extension (e.g., myprogram.txr, data_processor.txr). This standardized extension facilitates integration with development tools, version control systems, and build automation frameworks across your organization.

General Usage Pattern

The Tyxar CLI follows a consistent, predictable command structure: 

tyxar <command> <file> [options]
  • <command> – The operation to perform (run, build, convert, etc.)
  • <file> – Your Tyxar source file with .txr extension
  • [options] – Optional flags to customize command behavior

Complete Command Reference

The Tyxar CLI provides a comprehensive set of commands designed for maximum efficiency and intuitive discoverability. Each command is available in both a full and abbreviated form to accommodate your workflow preferences:

  • Help & Information – tyxar --help (or -h)
  • Version Management – tyxar --version (or -v)
  • Program Execution – tyxar run (or r)
  • Code Transpilation – tyxar convert (or c)
  • Executable Compilation – tyxar build (or b)
  • Package Installation – tyxar install (or i)
  • Static Code Analysis – tyxar scan (or s)
  • Code Explanation – tyxar explain (or e)

Getting Help: --help / -h

The integrated CLI help system provides immediate access to command reference information, usage examples, and available options without requiring external documentation. This feature is invaluable when you need quick clarification on command syntax or available flags.

Usage: 

tyxar --help
tyxar -h

The help command displays comprehensive information about all available Tyxar commands, their purpose, available options, and practical usage examples.

Version Information: --version / -v

Verify the currently installed version of the Tyxar programming language and CLI toolchain. Version information is crucial for compatibility verification, update planning, and troubleshooting compatibility issues between your development environment and project requirements.

Usage: 

tyxar --version
tyxar -v

Returns version information in a standardized format, including major, minor, and patch version numbers.

Program Execution: run / r

Execute your Tyxar programs immediately with the run command. This command interprets and executes your .txr files without requiring a compilation step, enabling rapid development iteration and immediate feedback on code execution.

Basic Usage: 

tyxar run myprogram.txr
tyxar r myprogram.txr

Advanced Execution Modes: 

tyxar run myprogram.txr --jit
tyxar run myprogram.txr -j

tyxar run myprogram.txr --ast
tyxar run myprogram.txr -a

Options:

  • --jit (or -j) – Enable Just-In-Time compilation for significantly faster execution performance, ideal for performance-critical applications.
  • --ast (or -a) – Run in Abstract Syntax Tree mode for detailed program analysis and debugging, useful when investigating program behavior or compiler interactions.

Code Transpilation: convert / c

Transpile your Tyxar source code into alternative high-level programming languages. This capability enables seamless integration with existing ecosystems, cross-platform compatibility, and interoperability with projects written in other languages.

General Usage: 

tyxar convert myprogram.txr --target py
tyxar convert myprogram.txr -t py

Supported Target Languages:

  • py – Python
  • js – JavaScript
  • c – C
  • ast – Abstract Syntax Tree dump

Examples: 

tyxar convert myprogram.txr --target py
tyxar convert myprogram.txr --target js
tyxar convert myprogram.txr --target c
tyxar convert myprogram.txr --target ast

Executable Compilation: build / b

Compile your Tyxar source code into optimized, standalone executable files suitable for distribution and deployment in production environments. This command generates platform-specific binaries or cross-platform artifacts depending on your target platform selection.

General Usage: 

tyxar build myprogram.txr --target exe
tyxar build myprogram.txr -t exe

Supported Build Targets:

  • exe – Windows executable
  • wasm – WebAssembly for web browsers
  • apk – Android application package

Examples: 

tyxar build myprogram.txr --target exe
tyxar build myprogram.txr --target wasm
tyxar build myprogram.txr --target apk

Package Installation: install / i

Install reusable code modules and extensions from online Tyxar repositories. Tyxar Nodes are pre-built, tested packages that provide extended functionality, third-party integrations, and community-contributed libraries.

General Usage: 

tyxar install package_name
tyxar i package_name

Package Management Options:

  • --list (or -l) – Display all currently installed nodes and their versions
  • --remove (or -r) – Uninstall a previously installed package from your environment

Example: 

tyxar install mypackage

Static Code Analysis: scan / s

Perform comprehensive static analysis on your Tyxar source code without executing the program. This powerful feature enables early error detection, code quality assessment, and potential issue identification before runtime execution.

Basic Usage: 

tyxar scan myprogram.txr
tyxar s myprogram.txr

Verbose Analysis: 

tyxar scan myprogram.txr --verbose
tyxar scan myprogram.txr -v

Options:

  • --verbose (or -v) – Generate detailed, granular feedback about identified issues, including severity levels, line numbers, and suggested corrections.

Code Explanation: explain / e

Generate comprehensive, human-readable explanations of what your Tyxar code does. This feature is invaluable for code comprehension, knowledge transfer, and educational purposes. The Tyxar explain command provides insights into program logic and execution flow without requiring manual code review.

Basic Usage: 

tyxar explain myprogram.txr
tyxar e myprogram.txr

Step-by-Step Analysis: 

tyxar explain myprogram.txr --steps
tyxar explain myprogram.txr -s

Options:

  • --steps (or -s) – Provide step-by-step breakdown of code execution, showing variable assignments, conditional branches, and control flow in sequence.

Pro Tips for CLI Usage

  • Use abbreviated commands – All commands have short forms (-r instead of run, -b instead of build) for faster typing during development.
  • Leverage the scan command – Run tyxar scan frequently to catch errors before they become runtime issues.
  • Use explain for learning – The explain command is an excellent tool for understanding program logic and team knowledge transfer.
  • Target multiple platforms – Use the build command with different targets to create platform-specific distributions.
